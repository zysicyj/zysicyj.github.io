有几项管理活动仅特定于 Space On-Premises：

  1. 上传 Space 产品许可证。

  2. 为移动客户端启用推送通知。

否则，在管理方面，Space On-Premises 与其云版本没有什么不同 - 有关基本管理说明，请参阅 https://www.jetbrains.com.cn/en-us/help/space/administrator-s-guide.html。

激活 Space On-Premises 实例

  1. 在浏览器中，打开干净的 Space On-Premises 安装。如果安装了 Docker Compose，则默认 URL 为 。http://127.0.0.1:8084

  2. 在“许可证”字段下，单击链接以登录您的 JetBrains 帐户。

  3. 在您的帐户页面上，复制许可证字符串并返回 Space。

  4. 将字符串粘贴到“许可证”字段。

  5. 如果需要，请填写其他字段，然后单击开始使用。

为移动客户端启用推送通知

Space 提供 适用于 iOS 和 Android 的移动客户端。要向移动客户端发送推送通知，您的 Space On-Premises 实例必须使用 JetBrains 的代理服务器。代理服务器通过相应的 Apple 和 Google 服务将通知发送给最终用户。

warning

通过启用代理服务器，您明确同意向 JetBrains 发送通知数据。

为代理服务器启用推送通知

  1. 确保您的 Space 实例有权访问 。https://on-premises.service.jetbrains.space

  2. 在主菜单上，单击“管理”
    ，然后选择“通知代理”。

  3. 选择 JetBrains Proxy，然后单击保存。

配置邮件服务器

Space On-Premises 没有邮件服务器。这意味着，默认情况下，Space 将无法发送电子邮件，例如用户邀请、聊天通知等。

要启用 Space 电子邮件通知，您必须自行创建邮件服务器并将其设置提供给 Space。通常，您可以在安装期间或以后在空间管理 UI 中提供设置。对于 AWS 中的 Docker Compose 安装，Space UI 是唯一的选项。

请注意，Space UI 中指定的邮件服务器设置会覆盖安装期间指定的设置。

  1. 在主菜单上，单击“管理”，然后选择“
    SMTP 设置”。

  2. 指定邮件服务器设置：

    • 加密：邮件服务器支持的加密类型（STARTTLS、SSL/TLS 或 None）。

    • 主机和端口：邮件服务器的主机名和端口号。

    • 用户名和密码：用于访问邮件服务器的凭据。

    • 电子邮件“发件人”地址：将用作空间通知发件人地址的电子邮件地址。

  3. 点击保存。
